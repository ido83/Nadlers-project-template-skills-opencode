# Frontend

Use this skill for browser-facing application work: pages, components, styling, layout, forms, routing, client state, accessibility, and user experience.

## Workflow

- Inspect existing design system, component conventions, styling approach, and state patterns before editing.
- Preserve the app's visual language unless the user asks for a redesign.
- Build responsive behavior for desktop and mobile.
- Prefer semantic HTML and accessible interactions over custom behavior.
- Keep client state as local as practical; avoid adding global state unless the existing app already uses it for the same concern.

## Checks

- Verify keyboard navigation, focus states, labels, color contrast, and screen-reader semantics for interactive changes.
- Verify loading, empty, error, disabled, and success states for async UI.
- Verify layout at narrow and wide viewport sizes.
- Run the relevant frontend lint, typecheck, test, or build command when available.

## Response Style

- Mention notable UX and accessibility considerations.
- Include any test or build command results.
